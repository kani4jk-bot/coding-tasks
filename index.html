Full policy

1. Information the app currently handles

Audio you choose to submit

When you record a clip or upload an audio file for identification, the app sends that audio to the backend API so the bird-identification service can analyze it. This can include the audio itself, file name, file type, file size, and request metadata.

Optional location context

You can choose whether to attach location context. In the web app, location can be entered manually or requested from the browser. In the native app, location is opt-in and currently converted to an approximate region of about 3 km² before upload instead of sending a precise pin.

Optional date context

The app may send a recording date to help narrow likely species. In the web app this is optional. In the native app, the device date is on by default but can be turned off.

Local history and notes

The native app stores successful identification results, saved/starred sightings, notes you add, and retry-queue items locally on your device. This local history is device-only right now.

2. What the app does not currently do

  • No user accounts
  • No cloud sync of saved sightings or notes
  • No cross-device history sync
  • No always-on listening or background audio surveillance
  • No precise live location tracking in the native app

3. How information is used

  • To process bird-identification requests
  • To improve matching with optional context like approximate location and date
  • To return results in the app
  • To keep local history, notes, and retry items on your device
  • To support basic operational debugging when requests fail

4. Storage and retention

On your device, local history, notes, stars, and queued retry clips stay there until you delete them or clear app data. Audio and request data are also sent to the backend for processing. Because this is an MVP, this page avoids making strong retention promises that have not been verified yet. Before publishing, confirm and document the real backend hosting, logging, and retention setup.

5. Permissions

  • Microphone access, to record a bird clip
  • Location access, only if you choose to attach location context

If you do not grant location access, the app can still work without location context.

6. Your choices

  • Choose whether to upload a clip for identification
  • Choose whether to attach location context
  • Turn date context off in the native app
  • Delete saved sightings and notes from the device
  • Clear local history and retry-queue items from the device

7. Changes

If the app later adds accounts, sync, analytics, crash reporting, or new service providers, this policy should be updated before or when those features ship.
