# Claude Design Prompt — Airtable-style UI

Use `DESIGN.md` as the primary visual/system brief for this project.

When designing or editing UI for this plant identification app:
- follow the Airtable-inspired principles in `DESIGN.md` first
- favor a bright, structured, utility-friendly interface with modular sections and clear information hierarchy
- make the app feel approachable and organized, with strong scanability for plant names, confidence, facts, and care tips
- prefer clean cards, tidy sections, purposeful accent color, and practical visual grouping over decorative flourish
- keep the photo upload and identification result flow fast and obvious
- preserve accessibility, responsive behavior, and implementation realism for the existing React/FastAPI app

For every UI task:
1. briefly state which Airtable design cues you are applying
2. keep changes modular, clear, and easy to scan
3. avoid drifting into rustic, overly playful, or overly dashboard-heavy styling unless explicitly asked
